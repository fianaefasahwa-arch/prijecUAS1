# Pendahuluan

Praktikum Ujian Akhir Semester (UAS) pada mata kuliah Bahasa Pemrograman ini bertujuan untuk menguji pemahaman mahasiswa dalam membuat program sederhana dengan menerapkan konsep modular programming dan Object Oriented Programming (OOP). Program yang dibuat harus mampu menerima input dari pengguna, melakukan validasi input, memproses data, serta menampilkan hasil dalam bentuk yang mudah dipahami.

# Tujuan Praktikum

Adapun tujuan dari praktikum ini adalah sebagai berikut:

1. Menerapkan konsep OOP dalam pembuatan program.

2. Memisahkan program ke dalam beberapa class (Data, Process, dan View).

3. Mengimplementasikan validasi input menggunakan konsep eksepsi.

4. Menampilkan hasil pengolahan data secara terstruktur.

# Implementasi Program

Program diimplementasikan menggunakan bahasa pemrograman Python. Alur kerja program adalah sebagai berikut:

1. Program meminta input dari pengguna.

2. Input divalidasi menggunakan mekanisme eksepsi.

3. Data diproses oleh class Process.

4. Hasil ditampilkan oleh class View dalam bentuk tabel sederhana.

Dengan pemisahan class tersebut, setiap bagian program memiliki tanggung jawab masing-masing.

# Hasil dan Pembahasan

Pada tahap hasil dan pembahasan, program yang dibuat telah berhasil dijalankan sesuai dengan ketentuan pada soal UAS. Program mampu menerima input dari pengguna melalui terminal, kemudian melakukan validasi input untuk memastikan data yang dimasukkan sesuai dengan tipe dan aturan yang telah ditentukan.
Jika pengguna memasukkan data yang tidak valid, program akan menampilkan pesan kesalahan (error message) yang informatif tanpa menghentikan proses program. Hal ini menunjukkan bahwa penerapan konsep eksepsi (exception handling) telah berjalan dengan baik dan membuat program lebih aman serta andal.

Selain itu, pemisahan class menjadi Data, Process, dan View membuat alur program menjadi lebih jelas. Class Data berfungsi sebagai penyimpan input, class Process bertanggung jawab dalam pengolahan data, sedangkan class View menampilkan hasil akhir dalam bentuk tabel sederhana. Dengan struktur ini, setiap class memiliki tanggung jawab yang spesifik sehingga memudahkan proses debugging dan pengembangan program di masa depan.

Hasil output yang ditampilkan sudah sesuai dengan input dan proses yang dilakukan. Tampilan hasil dalam bentuk tabel membantu pengguna untuk membaca dan memahami informasi yang dihasilkan oleh program. Secara keseluruhan, program telah memenuhi seluruh kriteria yang diminta dalam tugas UAS.

# Kesimpulan

Program sederhana dapat dibuat dengan menerapkan konsep OOP dan modular.
Pemisahan class Data, Process, dan View membuat program lebih terstruktur.
Validasi input menggunakan eksepsi sangat penting untuk meningkatkan keandalan program.
